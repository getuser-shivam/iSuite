## 🌐 **OVERVIEW**

iSuite is a **completely free, cross-platform application suite** built with Flutter and PocketBase, featuring **advanced AI-powered file management** and **comprehensive network sharing** capabilities. It provides enterprise-grade functionality with **zero cost** and **complete parameterized configuration**.

### ✨ Key Features

- 🤖 **AI-Powered Intelligence**: Smart file organization, semantic search, duplicate detection, and personalized recommendations
- 🌐 **Cross-Platform Support**: Native performance on Android, iOS, Windows, Linux, macOS, and Web
- 💰 **Completely Free**: Zero cost with free hosting options for all platforms
- 🔧 **Parameterized Configuration**: Centralized configuration with environment overrides
- 🚀 **High Performance**: Multi-level caching, parallel processing, and optimized algorithms
- 🛡️ **Privacy-First**: All AI processing happens locally with no external dependencies
- 📊 **Advanced Analytics**: Real-time performance monitoring and usage insights
- 🎯 **Smart Organization**: AI-driven file categorization and automated directory structuring
- 🔍 **Intelligent Search**: Semantic, fuzzy, and auto-complete search with learning capabilities

- 🔐 **Enterprise Security**: End-to-end encryption, secure authentication, and access control
- 📱 **Device Discovery**: Automatic network device discovery and connection management
- 🔄 **Cross-Protocol**: Seamless file transfers across different network protocols

## 🏗️ **Architecture**

### **5-Level Component Hierarchy**

```
Level 5: AI Services (Document Intelligence, Search, Translation, Version Control)
├── Level 4: Robustness Services (Circuit Breaker, Health Monitor, Data Validation, Performance)
├── Level 3: Core Services (CentralConfig, Logging, Security, Build Optimization)
├── Level 2: Business Services (File Ops, Network, Cloud Storage, Analytics, Local Sharing)
└── Level 1: Infrastructure (PocketBase, Supabase, SQLite, Platform APIs)
```

### **Clean Architecture Implementation**

- **Presentation Layer**: UI components with Riverpod state management
- **Domain Layer**: Business logic and models
- **Data Layer**: Repositories and services
- **Infrastructure Layer**: External APIs and platform-specific code

### **Centralized Parameterization**

All components are parameterized through `CentralConfig` singleton:
- UI parameters (colors, spacing, typography)
- Service settings (timeouts, buffer sizes, retry logic)
- Feature toggles and behavior customization
- Real-time parameter updates without app restart

### Configuration

#### Central Configuration System
iSuite uses a centralized parameter system for all settings:

- **UI Parameters**: Colors, fonts, spacing, animations
- **Feature Flags**: Enable/disable features

- **AI Configuration**: API keys, model settings

- `SUPABASE_URL`: Supabase project URL

## Troubleshooting

### Common Issues

**Flutter Doctor Warnings:**
```bash
flutter doctor --android-licenses
flutter doctor --ios-licenses
```

**Build Errors:**
- Clear cache: `flutter clean && flutter pub get`
- Update dependencies: `flutter pub upgrade`
- Check Flutter version compatibility

**Network Issues:**
- Verify internet connection
- Check firewall settings
- Validate API keys in `.env` file

**AI Features Not Working:**
- Verify GLM API key in settings
- Check internet connectivity
- Ensure API key has sufficient credits

### Debug Mode
Enable debug features in settings for additional logging and error reporting.
